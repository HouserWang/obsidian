### mmap
* **流程*： Disk -> Page Cache <--(Mapping)--> User Space
* ***场景** ：高性能读写
* ***Java 对应：** MappedByteBuffer。
## sendfile
- **流程 ：Disk -> Page Cache --(DMA Copy)--> NIC
- **场景**：适合**大文件顺序传输**，本质是**数据不进用户态**。你无法修改数据，只能转发。
- **java对应**：FileChannel.transferTo()
## Direct I/O
- **流程 ：Disk <--(DMA)--> User Buffer
- **场景**：mysql需要精细化操作缓存，**bufferpool**代替
- **java对应**：JDK 原生支持较弱，通常需要通过 JNA/JNI 调用 Native 方法打开文件（open with O_DIRECT），或者使用 Netty 的 Native Transport 封装。
