实际流程：Young GC循环 → 达到IHOP → Young GC+初始标记 → 并发标记+最终标记+清理 → 多次Mixed GC → 回到Young GC循环 ^7jkUB0q2

1. 标记阶段（Marking Phase）——并发标记存活对象

小阶段 1：根扫描（Root Scanning）
短暂 STW（<1ms），扫描线程栈、寄存器、JNI 全局引用等根对象。
小阶段 2：并发标记（Concurrent Marking）
应用线程运行时，GC 线程并发遍历对象图，标记存活对象。
使用着色指针（Marked0 或 Marked1）记录标记状态。
小阶段3 ： 再标记（Remark）：极短 STW（<1ms），专门处理弱根

2. 准备重定位阶段（Relocation Preparation Phase）——决定哪些对象要移动

小阶段 1：选择重定位集合（Relocation Set Selection）
根据标记结果，挑选垃圾最多的 region（稀疏 region），决定要搬移哪些存活对象。
小阶段 2：并发重定位准备（Concurrent Relocation Preparation）
把要搬移的对象复制到新 region（新物理页），同时把新物理页映射到 Remapped View。
旧物理页继续映射到 Marked0/Marked1 View。

3. 重定位阶段（Relocation Phase）——核心并发转移阶段

小阶段 1：并发重定位（Concurrent Relocation）
应用线程运行时，每次读指针都会触发加载屏障（Load Barrier）。
屏障检查指针颜色：
如果是 Marked → 立刻把指针修正为 Remapped View 地址，并设置 Remapped 颜色。
如果是 Remapped → 直接使用（已安全）。
这样所有旧指针逐步被应用线程自己修正，无需 STW。


4. 清理阶段（Cleanup Phase）——回收垃圾

小阶段 1：并发清理（Concurrent Cleanup）
GC 线程回收不再被任何指针引用的旧 region（物理页），并把内存归还 OS（uncommit）。
小阶段 2：短暂 STW 结束
极短 STW（<1ms），确认所有根和屏障都已完成，GC 周期结束。 ^Owibw8qh

ZGC 最大的优势是利用着色指针 + 加载屏障 + 虚拟内存多视图映射，把 G1 中必须 STW 的对象复制（evacuation）阶段变成了完全并发，从而实现了与堆大小无关的亚毫秒级暂停 ^aM0Ho3oe
